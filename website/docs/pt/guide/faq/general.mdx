# Perguntas Frequentes Gerais

### Qual é a relação entre Rsbuild e Rspack?

Rspack é o empacotador base para o Rsbuild. O objetivo do Rsbuild é fornecer capacidades de construção prontas para uso para usuários do Rspack, permitindo que os desenvolvedores iniciem um projeto web com configuração zero.

As principais diferenças entre Rspack e Rsbuild são:

- Projetos Rspack precisam ser configurados do zero, enquanto o Rsbuild fornece configurações padrão de melhores práticas e suporta a extensão de configurações do Rspack.
- Projetos Rspack exigem integração com loaders e plugins da comunidade para suportar diferentes cenários, enquanto o Rsbuild fornece plugins oficiais e suporte padrão para frameworks frontend comuns e capacidades de construção.
- As capacidades da CLI do Rspack são comparáveis à CLI do webpack, com funcionalidade relativamente simplificada, enquanto o Rsbuild fornece uma CLI mais poderosa e um servidor de desenvolvimento mais completo.

---

### O Rsbuild pode ser usado para construir bibliotecas ou componentes de UI?

O Rsbuild é pronto para uso na construção de aplicações web.

Para bibliotecas e componentes de UI, é recomendado usar o [Rslib](https://github.com/web-infra-dev/rslib), que é uma ferramenta de desenvolvimento de bibliotecas baseada no Rsbuild, e pode reutilizar a configuração e os plugins do Rsbuild.

---

### Qual é a relação entre Rsbuild e Modern.js?

Modern.js é um framework de desenvolvimento web progressivo construído sobre o Rsbuild. As capacidades de construção do Modern.js são baseadas no Rsbuild.

As principais diferenças entre Modern.js e Rsbuild são:

- Modern.js é baseado em React, enquanto o Rsbuild não é acoplado a nenhum framework de UI frontend.
- Modern.js é uma solução full-stack, fornecendo capacidades de runtime e server-side, enquanto o Rsbuild é uma ferramenta de construção com outras capacidades extensíveis através de plugins.
- Modern.js tem mais recursos embutidos, enquanto o Rsbuild busca leveza e flexibilidade.
