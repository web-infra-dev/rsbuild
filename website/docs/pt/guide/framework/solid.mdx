# Solid

Neste documento, você aprenderá como construir uma aplicação Solid usando o Rsbuild.

## Criar uma aplicação Solid

Use o [create-rsbuild](/guide/start/quick-start#create-an-rsbuild-application) para criar uma aplicação Solid com o Rsbuild. Execute o seguinte comando:

import { PackageManagerTabs } from '@theme';

<PackageManagerTabs
  command={{
    npm: 'npm create rsbuild@latest',
    yarn: 'yarn create rsbuild',
    pnpm: 'pnpm create rsbuild@latest',
    bun: 'bun create rsbuild@latest',
  }}
/>

Em seguida, selecione `Solid` quando for solicitado "Select framework" (Selecionar framework).

## Usar Solid em um projeto existente

Para compilar componentes Solid, você precisa registrar o [plugin Solid](/plugins/list/plugin-solid) do Rsbuild. O plugin adicionará automaticamente a configuração necessária para builds do Solid.

**O que são Componentes Solid?**
Solid é uma biblioteca JavaScript declarativa para criar interfaces de usuário, conhecida por seu desempenho e reatividade fina. Diferente de React ou Vue, Solid não usa um Virtual DOM. Em vez disso, ele compila seus componentes (escritos em JSX/TSX) diretamente para nós DOM reais e atualiza apenas as partes do DOM que realmente mudaram, resultando em um desempenho muito eficiente. A reatividade em Solid é baseada em sinais (signals), que são valores reativos que notificam automaticamente os consumidores quando mudam.

Por exemplo, registre em `rsbuild.config.ts`:

```ts title="rsbuild.config.ts"
import { defineConfig } from '@rsbuild/core';
import { pluginBabel } from '@rsbuild/plugin-babel';
import { pluginSolid } from '@rsbuild/plugin-solid';

export default defineConfig({
  plugins: [
    pluginBabel({
      include: /\.(?:jsx|tsx)$/,
    }),
    pluginSolid(),
  ],
});
```
