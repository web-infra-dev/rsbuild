# server.base

- **类型：** `string`

- **默认值：** `/`

`server.base` 用于配置服务端的[基础路径](/guide/basic/server#服务端基础路径)。

### 示例

默认情况下，服务端的基础路径为 `/`，用户可通过 http://localhost:3000 访问到 `index.html`。

当我们希望通过 http://localhost:3000/foo 访问到 `index.html` 时，可将 `server.base` 修改为 `/foo`。

```ts
export default {
  server: {
    base: '/foo',
  },
};
```

### 修改静态资源前缀

需要注意的是，该配置并不会修改前端生成的静态资源前缀。

为了保证本地开发时，静态资源能够通过 Rsbuild 开发服务器正常访问到，[dev.assetPrefix](/config/dev/asset-prefix) 应当和 `server.base` 包含相同的 url 前缀：

```ts
export default {
  dev: {
    assetPrefix: '/foo/',
  },
  server: {
    base: '/foo',
  },
};
```

同理，如果你希望静态资源能够在 Rsbuild 预览 ([rsbuild preview](/guide/basic/cli#rsbuild-preview)) 时正常访问到，[output.assetPrefix](/config/output/asset-prefix) 应当和 `server.base` 包含相同的 url 前缀：

```ts
export default {
  output: {
    assetPrefix: '/foo/',
  },
  server: {
    base: '/foo',
  },
};
```
